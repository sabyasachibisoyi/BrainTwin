🖥️ Chrome Extension Built

Runs on every page. Tracks dwell time (30s threshold) and POSTs to /capture when you've actively read for the full window. Popup shows today's count; Pause/Resume toggle backed by chrome.storage.local so it survives the MV3 service worker dying. Skip-list excludes banking and auth domains.

📱 Telegram Bot Built

Mobile capture inbox. Share / forward anything from any phone app to the bot. Acks within ~1s with 📥 Captured; replies inline with ⚠️ Couldn't process on failure. Same POST /capture contract as the extension. Runs as a separate process via python -m backend.telegram_bot.bot; allowlist auth via ALLOWED_TELEGRAM_USER_IDS; long-polling so no public URL needed. Decision 3: drops your interpretive layer (typed thoughts, your own captions on forwards) so the system has to infer connections from observed consumption alone.

Phase 3 — sentence-transformers

🏷️ LLM Enrichment Output

Every successfully captured row gets a 4-field block: a 1–2 sentence summary, entities (person / org / place / event, transliterated to Latin script), key_facts (atomic claims with numbers, dates, names), and topics (3–5 lowercase hyphenated tags). Persisted as a sidecar data/enrichments.jsonl, joined to captures.jsonl by capture_id. Runs async via FastAPI BackgroundTasks — /capture never waits on Haiku. Failures log to capture_failures.jsonl with phase: "enrichment" and surface in /failures.

Claude Haiku~$0.003/capture4-field schemaasync BackgroundTasks

🪡 Capture Hydration Tiers Phase 2.5 LIVE

Closes the gap surfaced by Phase 2 smoke testing — IG reels and FB share links forwarded via Telegram landed as empty_content failures because the bot can only send the URL and the backend has no fetcher beyond YouTube. Phase 2.5 adds a tiered capture-time hydration: (1) use raw text if present, (2) use Telegram's link preview if present (free), (3) fetch og:title/og:description/og:image, (4) for video URLs run yt-dlp + local whisper.cpp, (5) tag phase: "enrichment_skipped" so genuine failures stop drowning in noise.

Phase 4

🤖 Agent Layer (RAG) Not built

🔍 Retrieval

When a question comes in, semantic search finds the top 20 relevant knowledge chunks from ChromaDB. Entity search pulls structured data from SQLite. Results are merged and deduplicated.

ChromaDB querySQLite lookupTop-K fusion

💭 Reasoning

Retrieved knowledge is injected into Claude Sonnet's context. The agent reasons across multiple pieces of knowledge to make connections — just like connecting "Kingfisher" to "Deepika Padukone" to "Vijay Mallya fled India."

Claude Sonnet~$0.01/query

📋 System Prompt
